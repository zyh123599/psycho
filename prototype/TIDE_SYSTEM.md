# V0.4 四股潮向、日报与对话系统

## 产品目标

四股潮向为 Reigns 式选择提供即时、可见的游戏反馈；潮笺让一次满潮变成可回顾的短句；日报和对话则把这些体验接回每天的生活。它们共同服务于“观察当下”，不构成心理测评、健康指标、疗效指标或长期人格画像。

## 四个维度

| 潮向 | 含义 | 满潮短句语气 |
| --- | --- | --- |
| 照见 `◐` | 辨认念头、感受和正在发生的模式 | 观察、命名、拉开一点距离 |
| 安住 `⌁` | 为身体、边界和当下留出落脚处 | 暂停、允许、回到此刻 |
| 相连 `∞` | 与他人，也与内在不同声音保持联系 | 真实表达、被听见、内在对话 |
| 余力 `✦` | 照顾当下可用的力气，也允许疲惫存在 | 小步行动、保留力气、不必耗尽 |

这些名称是叙事隐喻，不应被替换为“心理健康、稳定度、社交能力、执行力”等易被理解为评估结果的标签。

## 章节潮位规则

- 每章四条潮位从同一基线开始，只在本次页面内存中存在。
- 每个左右选项推动一条主要潮向和一条次要潮向；「两个都不像」也会推动照见与安住，因为纠正选项本身是一种觉察与停顿。
- 选项按钮只提前显示会被牵动的潮向，不显示后台增量数字。
- 前端可以展示无数字进度条和满格状态；不得在结束页展示四维精确值、总分、排名、长短板或雷达图。
- 潮位达到隐藏阈值后以前端满格呈现，并触发该潮向的原创短句卡。每个潮向每章最多触发一次。
- 满潮不是答对、升级、治疗进展或“优势维度”；没有低潮惩罚、失败状态和最优路线。
- 新开一章或刷新页面后潮位清空，不写入 `localStorage`，也不直接进入长期画像或日报。

## 满潮、潮笺与卡槽

满潮后的两个动作含义应清晰区分：

| 动作 | 本章是否继续 | 是否放入卡槽 | 后续影响 |
| --- | --- | --- | --- |
| 收进卡槽，继续 | 是 | 是 | 本机保存内置卡片 ID 与收藏时间；该短句可成为本次未来回响候选，也可在结束后回顾 |
| 让它经过 | 是 | 否 | 不保存，不影响后续潮位、对话、微行动或完成结果 |

补充规则：

- 使用「来到潮面」而非「达成、满级、最强维度」等成就语言。
- “收进卡槽”是明确收藏，不是对某种心理特征的确认；“让它经过”也不是放弃或错失奖励。
- 卡槽只保存内置卡片 ID 与收藏时间，不保存短句副本、四维向量、选择、原始闪念、聊天或用户画像。
- 同一卡片自动去重；用户可在独立卡槽和章节完成页回顾，也可在「我的」中全部清除。
- 未来回响是另一层主动选择：只有最终勾选保存后，所选文字与解封日期才写入本机。
- 短句须原创或完成版权核验；避免保证康复、浪漫化痛苦、归因、训诫和伪装成预言。

## 与情绪陪伴对话的关系

V0.4 用开放式对话替代固定“三轮回应”，但不把聊天变成没有边界的“AI 治疗”。

- 完成六张情境卡后，主线自动进入对话；自由对话也可从首页、底部导航或日报单独进入。
- 章节开场只承接用户已确认的本次主题和少量结构化选择汇总，例如“为暂停或一小步留过位置”；不需要发送原始闪念、完整选择轨迹或四维数值。
- 对话不设必须完成的轮数，用户可以纠正、跳过、只想被听见或随时结束。
- 对话内容默认不改变四股潮位，不触发满潮，不授予潮笺，也不成为奖励或连续使用条件。
- 聊天原文默认不进入长期画像，也不用于生成日报。若未来要提取摘要信号，必须单独说明用途、获得明确授权，并允许查看、纠正和删除。
- 用户表达即时危险或主动点击紧急帮助时，应暂停普通陪聊和游戏化反馈，优先展示经地区配置核验的现实支持路径。
- 自动安全级别只是界面路由信号，不是诊断，也不能证明“用户没有风险”。生产规则见 [BACKEND_CONTRACT.md](./BACKEND_CONTRACT.md)。

## 与今日潮汐日报的关系

当前 V0.4 原型采用最小信号范围：日报只根据用户**主动收进卡槽的潮笺类型**选择内容变体。

- 卡槽为空或个性化授权关闭时显示“基础日报”，不假装已经了解用户。
- 个性化日报可以说“近 30 天主动收藏了安住潮笺”，不能说“你是缺乏安全感的人”或“你的安住得分很低”。
- 日报包含一句今日判断、两条可解释依据、三条低负担建议和一句原创短句；建议应允许跳过，不形成任务压力。
- 日报不得读取原始闪念、聊天原文、隐藏潮位、危机表达或未主动收藏的满潮卡。
- 当前原型使用滚动 30 天窗口：选择收藏数量较多的潮笺类型，数量相同时以最近一次收藏决定内容变体。生产版可在经过验证后加入衰减规则，但必须继续向用户解释窗口和信号来源。
- 内部可使用排序权重，但前端不显示分数、百分比、置信度、健康结论或人格标签。
- 「有一点帮助 / 不太像我」是对**这份内容**的反馈，不是用户自评得分。“不太像我”应降低相关推断权重，不能被解释成用户抗拒、病识感不足或不配合。
- 日报反馈不得反向改变本章潮位，也不得触发奖励、惩罚或危机判断。

## 信号隔离原则

| 数据 | 可用于本章潮位 | 可用于本章对话开场 | 可用于日报 | 默认长期保存 |
| --- | --- | --- | --- | --- |
| 情境选择 | 是 | 仅结构化汇总 | 否 | 否 |
| 四维潮位 | 是 | 否 | 否 | 否 |
| 已确认主题 | 否 | 经本次授权后可用 | 否 | 否 |
| 本章原始闪念 | 否 | 默认不用 | 否 | 否 |
| 主动保存的随手闪念 | 否 | 仅在用户点「用它开始梳理」后复制到本章 | 否 | 本机保存至用户删除 |
| 聊天原文 | 否 | 仅当前会话上下文 | 否 | 否 |
| 主动收藏的潮笺类型 | 否 | 默认不用 | 是 | 本机保存至用户删除 |
| 日报反馈 | 否 | 否 | 经授权后只调内容权重 | V0.4 原型不保存 |
| 主动保存的未来回响 | 否 | 否 | 否 | 本机保存至用户删除 |

任一跨列复用都必须有新的产品理由、单独授权和删除路径，不能因为数据“已经存在”就默认扩大用途。

独立「闪念」入口是记录工具，不是隐形画像入口。保存便签只代表用户希望在本机回看；不得因此自动生成主题、日报、聊天上下文或心理标签。用户主动点「用它开始梳理」时，也只复制该便签前 120 字到当前章节，原便签仍由用户单独管理。

## 内容与安全验收

- 左右选项都应像真实的人会做出的选择，并各自具有保护功能；不能一边明显代表“健康正确答案”。
- 四维反馈只增加叙事感，不出现在章节结束页的分数、排名、人格标签或优劣结论中。
- 日报的每条个性化判断都有用户可理解的来源；数据不足时明确降级为基础版。
- 对话先尊重“只想被听见 / 不要建议”，避免连续追问、道德说教、确定性因果解释和制造依赖。
- 短句、日报和聊天不得宣称诊断、治疗效果或对用户心理状态作确定判断。
- 危机支持不受付费、次数、登录、章节进度或奖励机制阻挡。
- 正式上线前完成心理专业、内容安全、隐私合规、未成年人保护、地区危机资源维护与无障碍专项评审。
