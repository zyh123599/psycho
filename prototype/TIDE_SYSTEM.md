# 心潮 V0.6 潮向、潮笺与画像边界

四股潮向为选择提供即时叙事反馈；潮笺把一次满潮变成可回顾短句。它们服务于“观察当下”，不是心理测评、健康指标、疗效指标或人格画像。

## 四股潮向

- **照见**：辨认念头、感受与正在发生的模式；
- **安住**：为身体、边界与当下留出落脚处；
- **相连**：与他人及内在不同声音保持联系；
- **余力**：照顾当下可用的力气，也允许疲惫存在。

每章从固定本地潮位开始。左右选择和“两个都不像”只触发 `prototype/app.js` 中固定的 `tideEffects`；模型不能修改数值、阈值、排名或奖励。潮位只存在当前流程内存，刷新或新开一章后清空。

## 模型叙事与本地规则

自定义模型可以结合确认主题与本机多模态文字画像生成六张卡的角色、场景、问题和选择文案。六个位置的选择语义由本地预先约束，实际信号和潮位仍由本地代码附加。

模型生成失败、未授权或网络不可用时使用六张内置卡。用户开始答题后，迟到的异步模型响应不得替换正在阅读的卡片。

## 潮笺存储

当某股潮向首次满潮时，用户可以主动把一张内置短句收进卡槽。本机只保存：

```json
{
  "id": "grounding-1",
  "collectedAt": 1784361600000
}
```

不保存短句副本、便贴、选择、潮位、聊天或画像。同一卡片自动去重，用户可在「我的」中清空。

## 日报

- 未启用持续画像时，日报最多读取近 30 天主动收藏的内置潮笺类型；
- 启用持续画像后，日报可使用本机结构化画像中的标题、需要、资源、行动和反思问题；
- 图片只通过模型形成文字化多模态观察，日报不读取或重新上传原图；
- 日报不显示分数、排名、人格结论、诊断或“变好/变差”。

## 对话与画像

聊天原文只存在本次页面内存，并按次直达用户自定义服务商。用户单独开启持续画像后，近期用户消息可作为新的 `fresh_evidence` 触发后台画像更新；本机只保存模型返回的脱敏结构化文字结果，不保存聊天原文。

模型服务授权不等于画像授权。撤销持续画像会终止新的画像请求；删除画像会取消在途请求并清除本机画像，但不会自动删除用户独立保存的潮笺和未来回响。

## 安全边界

- 潮向、卡片和画像都不得被表述为诊断、治疗、人格判断或正确答案；
- 不从脸、表情、身体、衣着或外貌推断心理状态；
- 发现自伤、伤人或即时危险线索时，暂停游戏化普通内容，优先显示所在地紧急服务、危机支持资源与可信任真人支持；
- 完整自定义模型接口与 CORS 要求见 [`CUSTOM_API.md`](CUSTOM_API.md)。
