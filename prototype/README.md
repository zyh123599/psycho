# 心潮 V0.7：实时语音、纯前端多模态画像与画像混合叙事

当前版本没有应用后端。用户在「我的」中配置自己的 OpenAI 兼容 API，浏览器直接调用模型；持久化数据只写在当前设备的 `localStorage`。

## 运行与验证

在仓库根目录执行：

```bash
npm ci
npm run dev
npm run test:frontend
npm run build
```

OpenAI 兼容模型的 API Key 不要写入源码、`.env` 或构建参数，请在运行后的「我的 → 自定义模型 API」中保存。供应商兼容要求与结构化输出契约见 [`CUSTOM_API.md`](CUSTOM_API.md)。

实时语音转写复用现有麦克风入口。用户可在「我的 → 自定义语音转写」测试、保存和清除自己的讯飞 APPID、APIKey 与 APISecret；开发时仍可用 `.env.local` 作为回退配置。纯前端无法保护本机凭据或构建变量，详细边界见 [`REALTIME_ASR.md`](REALTIME_ASR.md)。

## 当前体验

底部导航为「今天 / 闪念 / 对话 / 我的」，潮笺卡槽与未来回响入口收在「我的」。

- **今日潮汐**：显示可解释依据、低负担建议和原创短句；数据不足时明确使用基础版。
- **答案之书**：每天从本地原创卡片抽取一条，不收集用户心里的问题。
- **随手闪念**：同一张便签可组合文字、图片与实时语音转写，并以堆叠卡组回看；只持久化用户确认的文字、日期和表达类型元数据，原始图片与音频不落盘。
- **雨声背景**：由用户主动播放或暂停本地音频，不绕过浏览器自动播放限制。
- **自由对话**：聊天麦克风与语音陪伴弹层实时转写到同一可编辑输入框；用户主动发送后，文字才进入对话与画像链路。
- **今日章节**：闪念与授权图片形成主题，模型结合最新本机画像异步生成六张叙事卡。
- **未来回响**：只有用户主动勾选后才保存文字和解封时间。
- **我的时光**：按月历聚合本机闪念与未来回响，可选日期查看当天片段，并可在授权后生成月度模型回顾。

未配置、未授权、CORS 失败、超时或模型输出无效时，答案之书、闪念、内置叙事、潮向和本地对话降级仍可使用。

## 持续多模态画像

图片不是单独保存的“画像附件”。模型在同一次请求中读取用户授权的文字与章节图片，返回结构化 `multimodal_observations`：

- 参与观察的图片/文字 source ID；
- 图片或跨模态的文字观察；
- 该观察对暂时性画像的贡献；
- 不确定性。

原始章节图片只存在当前页面内存。图片处理完成后，只有上述文字观察随画像写入本机，后续更新通过独立的 `previous_profile_context` 延续。语音原音频只在录音期间实时发送给讯飞，不会发送给画像模型；用户确认提交的转写文字会标为 `voice_transcript`，并明确视为可能有识别误差的可修正证据。

独立闪念会写入 `xinchao.quick-notes.v1`，但持久化前会剔除图片 base64，且从不写入录音数据。记录只保留 ID、用户确认保存的文字（含语音转写）、创建时间、日历日期，以及 `hadImage`、语音时长等表达类型元数据；原图只在当前页面内存中供预览。图片只有在用户逐图确认处理权利，且持续画像已开启时才最多发送一次。

旧画像不会伪装成新的第一人称证据：新文本、互动和图片位于 `fresh_evidence`，旧画像只作为可被覆盖的连续性上下文。本地还会过滤“本次没有新增第一人称近况”等流水线元话术。

## 画像驱动内容

- 未开启持续画像时，日报使用近期闪念文字主题与主动收藏的内置潮笺类型；
- 开启后，画像可驱动主题候选、初印象、日报、微行动、未来回响、对话表达和叙事卡；
- 模型只生成叙事文案，六个位置的选择语义、潮向、信号与满潮规则仍由本地固定；
- 图片不会被日报重新上传，也不会根据脸、表情、身体、衣着或外貌推断心理属性。

## 我的时光与月度回顾

「我的时光」不建立另一份云端档案。它实时读取 `xinchao.quick-notes.v1` 与 `xinchao.future-echoes.v1`，在本地生成月历：闪念落在创建日期；未来回响会标记留下日期和约定返回日期。尚未解封的回响在本机月历中只显示占位文案；发送月度任务时，无论是否解封，模型都只收到回响状态占位，不收到回响正文。

月度回顾分为两层：

1. 本地层始终可用，只整理当月有记录的日期、闪念数和未来回响数，不分析人格或心理状态；
2. 模型层不会因打开「我的」自动运行；用户主动点「生成」，且当月有片段、API 配置有效、AI 调用授权与持续画像均开启时才异步请求。请求最多携带所选月份最近的 30 个文字化片段；同一未来回响在模型输入中只出现一次。只有最新画像的 `generated_at` 属于同一月份时，才附带压缩后的画像上下文。

月度模型任务使用独立的 `xinchao_monthly_reflection` 严格 JSON Schema。响应包含 `analysis_status`、标题、摘要、最多三条观察、一个可跳过的问题、不确定性和安全提示；前端仍会做长度、数量和安全级别校验。安全级别异常时不缓存回顾，并打开产品边界提示。

成功结果写入 `xinchao.monthly-memory.v1`，最多保留 24 个月。缓存记录包含月份、证据指纹、生成时间、模型名和清洗后的结构化结果；指纹覆盖月份、当月片段和可选同月画像，证据不变时不会重复调用。点「更新」会绕过缓存强制重试。无片段、未授权、未配置、网络失败、超时、拒绝或结构无效时均保留本地日历和数量回顾。

## 异步与竞态

- 画像更新不阻塞闪念、主题、选卡、聊天、行动或完成页；
- 同一时刻最多运行一个画像请求，新变化合并到下一次快照；
- 同一时刻最多运行一个语音转写会话；切屏、关闭弹层、页面隐藏或取消会立即释放麦克风与 WebSocket；
- 本机只保存证据哈希，相同证据不会重复调用；
- 每张章节图片使用稳定 source ID，已处理图片不会在同一流程重复上传；
- 超过单次图片上限的已授权图片按异步队列分批处理；独立闪念图片还需逐图确认处理权利；
- 删除画像或撤销授权会取消在途请求，并阻止迟到结果重新写回；
- 删除闪念会先移除旧画像，再从剩余可用线索重建，避免摘要保留已删除来源；
- 用户开始答题后，迟到的叙事结果不会替换正在阅读的卡片。
- 切换月份、关闭 AI/画像授权或删除画像会取消月度回顾请求；请求序号防止迟到结果写到当前月份。

## 本地存储键

| 键 | 内容 |
| --- | --- |
| `xinchao.custom-api.v1` | 自定义 Base URL、API Key、模型名、图片精度 |
| `xinchao.custom-asr.v1` | 用户自定义的讯飞 APPID、APIKey、APISecret |
| `xinchao.ai-consent.v1` | AI 与持续画像授权开关 |
| `xinchao.reflective-profile.v1` | 内容版本 `2.0` 的最新多模态文字画像及证据指纹 |
| `xinchao.quick-notes.v1` | 闪念文字、创建时间、日历日期和表达类型元数据；不含图片或音频文件 |
| `xinchao.monthly-memory.v1` | 最多 24 个月的结构化模型回顾、证据指纹、模型名和生成时间 |
| `xinchao.tide-cards.v1` | 内置潮笺 ID 与收藏时间 |
| `xinchao.future-echoes.v1` | 用户主动保存的未来回响 |
| `xinchao.answer-book.v1` | 当天答案卡序号与日期 |
| `xinchao.report-feedback.v1` | 日报的低粒度反馈 |
| `xinchao.onboarding.v1` | 是否看过新手说明 |

闪念文字和日期会按上表持久化；闪念图片 base64、章节图片、聊天原文、未提交语音转写和章节选择只保留在当前页面内存。语音会调用麦克风并实时发送给讯飞，但不产生本机录音文件；保存闪念后只持久化转写文字、表达类型与时长元数据。画像键名保留 `v1` 是为了迁移已有数据，实际结构版本为 `2.0`。「我的时光」的月历选中状态不单独持久化，而是每次从闪念与回响记录重新派生。

## 隐私与安全事实

- 心潮自身不设置后端、账号或云端画像仓；
- 模型服务商仍会收到每次明确授权发送的文本、章节图片或对话，其保留政策由用户选择的服务商决定；
- 讯飞会在用户点击现有麦克风按钮并授权后收到实时音频；心潮不保存音频，讯飞侧保留与计费以其服务条款为准；
- 前端本地保存的 API Key 可被同源脚本、浏览器扩展或设备使用者读取，不适合公共设备；
- 聊天内置明确危险用语预检，模型也返回结构化安全提示，但本产品不能替代紧急服务；
- 画像、建议、初印象和叙事均是暂时、可纠正的反思材料，不是诊断、治疗或人格结论。

## 浏览器兼容

自定义模型服务商必须正确响应 CORS `OPTIONS` 预检，并允许 `Authorization`、`Content-Type`、`GET` 与 `POST`。实时语音需要 Web Crypto、WebSocket、AudioWorklet（或 ScriptProcessor 降级）与 `getUserMedia`；麦克风仅在 HTTPS、localhost 或受信任应用 WebView 中可用。HTTPS 前端不能调用 HTTP API；纯浏览器代码无法绕过这些安全边界。
