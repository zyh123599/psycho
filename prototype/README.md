# 心潮 V0.6：纯前端多模态画像与画像混合叙事

当前版本没有应用后端。用户在「我的」中配置自己的 OpenAI 兼容 API，浏览器直接调用模型；持久化数据只写在当前设备的 `localStorage`。

## 运行与验证

在仓库根目录执行：

```bash
npm ci
npm run dev
npm run test:frontend
npm run build
```

不要把 API Key 写入源码、`.env` 或构建参数。请在运行后的「我的 → 自定义模型 API」中保存。供应商兼容要求与结构化输出契约见 [`CUSTOM_API.md`](CUSTOM_API.md)。

## 当前体验

底部导航为「今天 / 闪念 / 对话 / 我的」，潮笺卡槽与未来回响入口收在「我的」。

- **今日潮汐**：显示可解释依据、低负担建议和原创短句；数据不足时明确使用基础版。
- **答案之书**：每天从本地原创卡片抽取一条，不收集用户心里的问题。
- **随手闪念**：同一张便签可组合文字、图片与语音演示，并以堆叠卡组回看；记录仅在当前页面内存中存在。
- **雨声背景**：由用户主动播放或暂停本地音频，不绕过浏览器自动播放限制。
- **自由对话**：支持快捷表达、文字与语音界面演示；文字可按次直达自定义模型服务商。
- **今日章节**：闪念与授权图片形成主题，模型结合最新本机画像异步生成六张叙事卡。
- **未来回响**：只有用户主动勾选后才保存文字和解封时间。

未配置、未授权、CORS 失败、超时或模型输出无效时，答案之书、闪念、内置叙事、潮向和本地对话降级仍可使用。

## 持续多模态画像

图片不是单独保存的“画像附件”。模型在同一次请求中读取用户授权的文字与章节图片，返回结构化 `multimodal_observations`：

- 参与观察的图片/文字 source ID；
- 图片或跨模态的文字观察；
- 该观察对暂时性画像的贡献；
- 不确定性。

原始章节图片只存在当前页面内存。图片处理完成后，只有上述文字观察随画像写入本机，后续更新通过独立的 `previous_profile_context` 延续。独立闪念中的语音是界面演示，不调用麦克风或上传音频。

旧画像不会伪装成新的第一人称证据：新文本、互动和图片位于 `fresh_evidence`，旧画像只作为可被覆盖的连续性上下文。本地还会过滤“本次没有新增第一人称近况”等流水线元话术。

## 画像驱动内容

- 未开启持续画像时，日报使用近期闪念文字主题与主动收藏的内置潮笺类型；
- 开启后，画像可驱动主题候选、初印象、日报、微行动、未来回响、对话表达和叙事卡；
- 模型只生成叙事文案，六个位置的选择语义、潮向、信号与满潮规则仍由本地固定；
- 图片不会被日报重新上传，也不会根据脸、表情、身体、衣着或外貌推断心理属性。

## 异步与竞态

- 画像更新不阻塞闪念、主题、选卡、聊天、行动或完成页；
- 同一时刻最多运行一个画像请求，新变化合并到下一次快照；
- 本机只保存证据哈希，相同证据不会重复调用；
- 每张章节图片使用稳定 source ID，已处理图片不会在同一流程重复上传；
- 删除画像或撤销授权会取消在途请求，并阻止迟到结果重新写回；
- 用户开始答题后，迟到的叙事结果不会替换正在阅读的卡片。

## 本地存储键

| 键 | 内容 |
| --- | --- |
| `xinchao.custom-api.v1` | 自定义 Base URL、API Key、模型名、图片精度 |
| `xinchao.ai-consent.v1` | AI 与持续画像授权开关 |
| `xinchao.reflective-profile.v1` | 内容版本 `2.0` 的最新多模态文字画像及证据指纹 |
| `xinchao.tide-cards.v1` | 内置潮笺 ID 与收藏时间 |
| `xinchao.future-echoes.v1` | 用户主动保存的未来回响 |
| `xinchao.answer-book.v1` | 当天答案卡序号与日期 |
| `xinchao.report-feedback.v1` | 日报的低粒度反馈 |
| `xinchao.onboarding.v1` | 是否看过新手说明 |

闪念原文、闪念图片预览、语音演示、聊天、章节选择和章节图片只保留在当前页面内存。画像键名保留 `v1` 是为了迁移已有数据，实际结构版本为 `2.0`。

## 隐私与安全事实

- 心潮自身不设置后端、账号或云端画像仓；
- 模型服务商仍会收到每次明确授权发送的文本、章节图片或对话，其保留政策由用户选择的服务商决定；
- 前端本地保存的 API Key 可被同源脚本、浏览器扩展或设备使用者读取，不适合公共设备；
- 聊天内置明确危险用语预检，模型也返回结构化安全提示，但本产品不能替代紧急服务；
- 画像、建议、初印象和叙事均是暂时、可纠正的反思材料，不是诊断、治疗或人格结论。

## 浏览器兼容

自定义服务商必须正确响应 CORS `OPTIONS` 预检，并允许 `Authorization`、`Content-Type`、`GET` 与 `POST`。HTTPS 前端不能调用 HTTP API；纯浏览器代码无法绕过这些安全边界。
