# 心潮 V0.4：今日潮汐与情绪陪伴原型

「心潮」把每日低负担阅读、Reigns 式叙事选择和可随时结束的情绪陪伴串成一段连续体验。四股潮向在章节中可见，但章节结尾不计算总分、排名、人格结论或“治疗效果”。

当前交付物是可在浏览器运行的移动端代码原型（HTML、CSS、JavaScript），不是 Figma 源文件，也尚未连接真实聊天或日报后端。

## 本地预览

在终端运行：

```bash
cd prototype
python3 -m http.server 4176 --bind 127.0.0.1
```

然后访问 [http://127.0.0.1:4176/](http://127.0.0.1:4176/)。停止预览时，在终端按 `Control-C`。

若要同时验证 Ionic + Capacitor 的 Web 构建，可在仓库根目录运行 `npm install` 和 `npm run dev`。

## V0.4 体验结构

### 首页与自由探索

- **今日潮汐**：先给出一句今日判断、两条可解释依据、三条低负担建议和一句原创短句。没有可用信号时明确显示“基础日报”。
- **随手闪念**：从首页或底部导航独立进入，最多写 200 字并明确保存到当前浏览器。可单张删除，或主动选择一张带入章节；不会自动进入日报、画像或聊天。
- **自由对话**：用户可以从首页或日报进入“潮伴”，使用快捷表达或自由输入，随时结束并返回首页。
- **今日章节**：保留完整叙事主线，适合愿意多花几分钟梳理一条具体线索的用户。

底部导航为「今天 / 闪念 / 对话 / 卡槽 / 我的」。未来回响仍可从到期提醒、章节完成流程和「我的」进入。

### 今日章节主线

1. **闪念便贴**：写下 2–4 个当下掠过脑海的念头，并决定它们是否参与本次梳理。
2. **主题确认**：系统提出临时候选；主题必须由用户确认或改写，不生成永久人格标签。
3. **情境选择与四股潮向**：通过六张 Reigns 式情境卡探索同一线索。选择会实时改变「照见、安住、相连、余力」四条无数字潮位。
4. **潮笺收集**：任一潮向满格后浮出一张原创短句卡。「收进卡槽，继续」会收藏内置卡片；「让它经过」不收藏。两种选择都继续主线，也都没有惩罚。
5. **情绪陪伴对话**：完成情境卡后进入开放对话。开场会承接本次已确认主题与少量结构化选择线索；用户可纠正、跳过、自由输入或立即结束。它替代 V0.3 的固定“三轮回应”。
6. **微行动**：从几件现实中可完成的小事里选择一步，也可以跳过；没有打卡和连续完成压力。
7. **未来回响**：本章收进卡槽的短句可成为候选，也可选择其他句子或自己写。只有主动勾选后，文字与解封日期才会保存到当前浏览器。
8. **完成页**：回顾本次主题、微行动、未来回响和已收藏潮笺，不展示四维分数、优劣判断或长期画像。

## 当前原型中的日报

- 日报只读取当前浏览器卡槽中用户近 30 天主动收藏的**内置潮笺类型**，不读取闪念、确认主题、情境选择、章节潮位或聊天原文。
- 当前算法选择近 30 天收藏数量较多的潮笺类型；数量相同时以最近一次收藏决定内容变体。窗口内没有记录时显示通用基础版。
- 日报会说明生成依据，并允许选择「有一点帮助 / 不太像我」。V0.4 中该反馈只改变当前页面提示，不上传、不持久保存，也尚未真正调整后续权重。
- 日报内容、建议和短句均为本地预设，不是 AI 对用户状态的判断。

## 当前原型中的对话

- “潮伴”目前使用本地规则和延时效果模拟回复，没有调用模型或远程服务。
- 聊天内容只存在于当前页面内存；结束或离开自由对话时立即清除，刷新或关闭页面也会清除，不写入 `localStorage`。章节对话只在本次主线内暂存，开始新章节后清除。
- 章节对话开场会使用本次已确认主题，以及“暂停或先走一小步”等选择的汇总；自由对话不读取章节内容，从日报进入时只使用“从日报聊起”的入口标记。
- 对话不限制固定轮数；用户可以随时点击结束。当前输入上限为 400 个字符。
- 原型包含关键词级危机演示和显式“我现在需要紧急帮助”入口。触发后普通陪聊会暂停，并提示联系所在地紧急服务、危机支持资源或可信任对象。它不是可直接上线的危机识别系统。

## 交互方式

- 点击或轻触按钮完成填写、确认和页面导航。
- 情境卡支持鼠标或触控左右拖动，也可直接点击左右选项。
- 使用键盘 `←`、`→` 完成左右选择，`↓` 选择「两个都不像」；可用 `Tab` 在可交互元素间移动。
- 安全说明等弹层可通过界面按钮关闭；支持时也可按 `Escape` 退出。
- 页面按移动端优先设计，也可在桌面浏览器中体验。

## 数据与隐私

- 章节中的闪念、确认主题、情境选择、章节潮位、聊天、微行动和未保存的未来回响只在当前页面内存中使用。
- 独立「闪念」入口中的便签只有点击「收好这张便签」后才写入浏览器 `localStorage`，最多保留 50 张；它不会自动进入日报、画像或聊天。只有点击「用它开始梳理」时，前 120 字才会复制进本次章节。
- 四股潮向在游戏中以无数字进度条展示；刷新页面或开始新章节后归零，不形成长期雷达图。
- 选择「收进卡槽」后，本机只写入内置潮笺编号和收藏时间；不保存短句副本、闪念、主题、选择、潮位、聊天或画像。同一卡片自动去重。
- 只有用户在「未来回响」中主动勾选保存时，回响文字和解封日期才写入浏览器 `localStorage`。
- 随手闪念、卡槽与未来回响均可在产品内清除；清理浏览器网站数据也会移除它们。
- 当前原型没有账号、云同步、服务端画像、数据上传或第三方模型调用。

生产接入所需的接口、数据分层、保留与删除规则见 [BACKEND_CONTRACT.md](./BACKEND_CONTRACT.md)。

## 产品与安全边界

“潮伴”的定位是**情绪陪伴和自我梳理伙伴**，不是“AI 治疗师”。卡片、四股潮向、日报和对话不能用于：

- 心理诊断、人格判定、风险证明或疗效评估；
- 替代心理咨询、医疗服务或现实中的人际支持；
- 保证用户会变好，或把满潮、收藏、连续使用包装成康复进度；
- 在用户表达即时危险时继续游戏化、奖励、泛化建议或普通角色扮演。

若用户或他人正处于即时危险，应暂停体验并联系所在地紧急服务、经核验的危机支持资源或能够立即到场的可信任对象。正式上线前仍需心理专业、危机安全、内容治理、隐私合规、未成年人保护和无障碍专项评审。
