# 心潮 V0.2：心理叙事代码原型

「心潮」把一个闪念，走成一次完整的自我梳理。它借用 Reigns 的左右选择形式，但不设置心理健康的输赢、诊断、评分或惩罚。

当前交付物是可在浏览器运行的移动端代码原型（HTML、CSS、JavaScript），不是已创建的 Figma 文件。

## 本地预览

在仓库根目录运行：

```bash
npm install
npm run dev
```

按照终端显示的地址打开页面。停止预览时，在终端按 `Control-C`。

## V0.2 完整主线

1. **闪念便贴**：写下 2–4 个当下掠过脑海的念头，并决定它们是否参与本次梳理。
2. **主题确认**：系统根据本次输入提出候选线索；主题必须由用户确认或改选，不生成永久人格标签。
3. **多章节选择**：通过 Reigns 式情境卡，从不同内在声音和生活角度继续探索同一条线索。
4. **三轮回应**：引用刚才的内容、提出一个澄清问题，并始终允许用户纠正、跳过或结束。
5. **微行动**：从几件现实中可完成的小事里选择一步，不用打卡，也没有连续完成压力。
6. **未来回响**：选择或写下一句话，决定是否主动保存在本机，并在未来重新看见它。

底部的「今天 / 回响 / 我的」入口用于返回主线、查看主动保存的内容，以及阅读数据与安全说明。

## 交互方式

- 点击或轻触按钮完成填写、确认和页面导航。
- 情境卡支持鼠标或触控左右拖动，也可直接点击左右选项。
- 使用键盘 `←`、`→` 完成左右选择；可用 `Tab` 在可交互元素间移动。
- 安全说明等弹层可通过界面按钮关闭；支持时也可按 `Escape` 退出。
- 页面按移动端优先设计，也可在桌面浏览器中体验。

## 数据与隐私

- 闪念便贴、确认主题、卡片选择、三轮回应和微行动只在当前页面的内存中使用；刷新或关闭页面后不会保留。
- 隐藏的四维信号只用于调整本次体验的节奏和内容，不展示精确分数、不生成长期画像，也不用于诊断。
- 只有用户在「未来回响」中主动选择保存时，回响文字和解封日期才会写入浏览器 `localStorage`。
- 本地回响只保存在当前浏览器和设备中，可在「回响」或「我的」中删除；清理浏览器网站数据也会移除它。
- 当前原型没有后端账号、云同步或数据上传。

## 非 AI 与安全边界

V0.2 使用预设内容和本地规则串联主题、卡片与回应，未接入大语言模型、远程 AI 服务或真实咨询师。页面中的回应只用于验证交互和叙事连续性，不应被理解为 AI 判断。

本产品原型不是心理测评工具，不提供心理诊断、心理治疗、医疗建议或危机干预，也不能替代专业帮助。若用户正处于危险或危机中，应联系当地紧急服务、危机支持资源或可信任的人。正式产品上线前仍需心理专业、内容安全、隐私合规与无障碍评审。

## 迁移到 Figma

当前应以代码原型为交互与内容基准。迁移到 Figma 时，可按六段主线及「今天 / 回响 / 我的」拆分 Frame，并把按钮、便贴、主题选项、情境卡、回应气泡、微行动和回响条目建立为可复用组件；颜色、字体、间距、圆角和动效时长应整理为变量或样式。

迁移完成后需要逐屏核对移动端尺寸、空状态、返回路径、数据说明和完整可点击流程。除非另有实际 Figma 文件链接，本仓库中的版本仍表示代码原型，而不是 Figma 源文件。
